# 방정식으로 사보타지하기

수업에서 사용하는 브라우저 기반 방정식 추리 게임입니다. React, Vite, TypeScript, 일반 CSS로 구성되어 있습니다.

## 실행 방법

```bash
npm install
npm run dev
```

개발 서버가 실행되면 `http://localhost:5173/`로 접속합니다. 이 프로젝트는 Vite 포트 `5173`을 고정해서 사용합니다.

## 화면 경로

- `/` : 시작 화면, 1학년 버전 / 3학년 버전 선택
- `/teacher` : 교사용 조작 및 게임 설정 화면
- `/display` : 학생용 HDMI 공개 화면

## 시작 흐름

1. 시작 화면에서 1학년 버전 또는 3학년 버전을 선택합니다.
2. `/teacher`에서 플레이어 수, 목표 승점, 바퀴 수, 사보타지 수를 입력합니다.
   - 1학년 권장값: 22명, 목표 승점 10점, 3바퀴, 사보타지 3명
   - 3학년 권장값: 28명, 목표 승점 10점, 2바퀴, 사보타지 4명
   - 플레이어 수는 5명 이상 40명 이하입니다.
   - 목표 승점은 1점 이상입니다.
   - 바퀴 수는 1바퀴 이상 5바퀴 이하입니다.
   - 사보타지 수는 1명 이상이며 플레이어 수보다 작아야 합니다.
3. `사보타지 배정 확인`을 누르면 교사용 확인 화면에서 사보타지 번호를 확인합니다.
4. 배정 확인 화면에서 플레이어 수, 바퀴 수, 사보타지 수, 목표 승점을 확인합니다.
5. 필요하면 `이전으로` 돌아가 설정을 수정하거나, `다시 배정하기`로 같은 조건에서 번호를 다시 뽑습니다.
6. `게임 시작`을 누르면 본 게임 진행 화면으로 넘어갑니다.

## 설정 기본값

- 1학년 버전: 플레이어 수 22명, 목표 승점 10점, 3바퀴, 사보타지 3명
- 3학년 버전: 플레이어 수 28명, 목표 승점 10점, 2바퀴, 사보타지 4명
- 기본값은 설정 화면에 자동 입력되지만, 실제 출석 인원과 수업 운영에 맞게 교사가 수정할 수 있습니다.
- 사보타지 번호는 입력한 사보타지 수만큼 무작위로 배정됩니다.
- `다시 배정하기`를 누르면 같은 플레이어 수와 사보타지 수로 번호만 다시 뽑습니다.

## 공개 정보 규칙

- 교사용 사보타지 확인 화면에서만 전체 사보타지 번호를 표시합니다.
- 본 게임 화면에서는 전체 사보타지 번호 목록을 표시하지 않습니다.
- 목표 승점, 바퀴 수, 사보타지 수는 게임 설정 화면에서만 입력하고, 본 게임 중에는 표시만 합니다.
- 본 게임 화면의 `내 역할 확인` 버튼은 현재 차례 학생의 역할만 잠시 표시합니다.
- 학생용 `/display` 화면에는 사보타지 번호, 역할 정보, 정답 근, 내부 계산값, 교사용 조작 버튼이 표시되지 않습니다.
- 게임 설정 중 `/display` 화면에는 `게임 설정 중입니다`만 표시됩니다.

## 1차 규칙 요약

- 카드 추가하기: 후보 식 카드 3장 중 1장을 골라 좌변 또는 우변에 추가합니다.
- 근 제시하기: 좌변과 우변에 카드가 있고 전체 카드 수가 4장 이상일 때 x=-5부터 x=-1, x=1부터 x=5 중 하나를 제시합니다.
- 근 제시 성공: 승점 +2점
- 근 제시 실패: 승점 -1점
- 항등식은 어떤 값을 골라도 성공입니다.
- 화면 공유 및 추리하기: 학생용 화면에 행동 기록을 공개하고, 교사용 화면에서 사보타지 의심 번호를 지목합니다.
- 모든 턴이 끝나면 현재 승점과 목표 승점을 비교해 일반 플레이어 승리 또는 사보타지 승리를 표시합니다.

## 타이머

- 카드 추가하기: 20초
- 근 제시하기: 20초
- 화면 공유 및 추리하기: 60초
- 시간 초과 시 아무 행동 없이 턴이 종료됩니다.
